OS第七章设备管理

发表于 2020-12-12 分类于 课程复习

第7章 设备管理

分类(按数据组织方式、固有属性的分类原则)

  • 按设备的数据组织分类
    • 字符设备
    • 块设备
  • 按设备的固有属性分类
    • 独占设备
    • 共享设备

设备独立性含义

  • 用户或程序中使用的设备与具体的物理设备无关，用户或程序使用的是逻辑设备，当进程运行时，由操作系统在逻辑设备与物理设备之间建立连接，即设备的分配，把这种设备使用方法的特点称为设备独立性

I/O控制方式

  • 程序查询方式
    • 在执行一个进程的I/O操作时，处理器循环测试设备状态，状态为空闲时，处理器向设备控制器提交I/O操作，之后不断检查设备的标志寄存器，只有检查发现I/O操作完成后才执行后续指令
  • 中断方式
    • 某一事件的出现，要求处理器暂停当前进程的运行，而转向处理出现的事件，处理器完成该事件后才能继续原来的进程
  • DMA方式
    • 对于块设备的I/O操作，采用中断方式时，处理器的利用率将受很大影响，采用DMA可以避免
    • DMA方式也称为直接存储器存取，通过硬件即DMA控制器，实现存储器与设备，设备与设备之间的数据传输，不需要处理器参与，可以实现大批量数据的快速传输
  • 通道方式
    • DMA传输速度快，减少了处理器中断次数，但是因为DMA与处理器共享系统总线，影响了处理器利用率
    • 对于大型计算机系统和现代微处理器，可以提供专用I/O处理机，即通道方式实现I/O操作

I/O缓冲含义与目的

  • 含义
    • 在设备的I/O操作中，利用一种存储部件或其中的部分，暂时存放要交换的数据，将来再把数据传输到目标位置，这种数据暂存的技术称为缓冲技术
  • 目的
    • 缓解设备和处理器之间速度不匹配的矛盾，提高系统工作的并行程度
    • 减少I/O操作的次数
    • 减少中断次数
    • 提高系统及时性，方便用户操作

设备分配相关数据结构

  • 系统设备表
  • 设备控制表
  • 控制器控制表
  • 通道控制表

磁盘驱动调度组成

  • 目的
    • 对于一组磁盘I/O操作的请求，系统按一定策略依次为各个I/O操作的请求服务，使得完成这组磁盘I/O操作所花的时间尽可能小
  • 由于同一个磁盘的每个I/O操作，存取一个物理块的时间大致相等，所以磁盘驱动调度时，通常不考虑传输时间T，并且把磁盘驱动调度分为移臂调度和旋转调度组成

移臂调度算法

  • FCFS（先来先服务）
    • 按I/O操作请求提出的时间先后顺序，调度时选择最先提出的一个I/O操作请求
    • FCFS移臂调度算法可能导致频繁改变磁头方向
    • 例如：假定某磁盘的一组I/O操作的访问提出的顺序依次是55、72、100、88、93、63（I/O操作请求所在的柱面号），当前磁盘位于90号。则服务顺序为90-55-72-100-88-93-63,移动距离35+17+28+12+5+27=124（跨越的柱面总数），其中磁头方向改变4次
  • SSTF（最短寻道时间优先算法）
    • 按各请求所在的柱面，调度时选择距离当前磁头最近柱面的一个I/O操作请求
    • 按上述例子，服务顺序为90-88-93-100-72-66-55，移动距离2+5+7+28+6+11=59，其中磁头改变方向2次
  • 扫描算法（SCAN）
    • 磁头从着陆区启动后，从内向外方向扫描移动，达到最外层柱面后改变方向从外向内扫描，在扫描移动过程中总是选择当前磁头方向上距离最近柱面的I/O操作，为其服务
    • 例：假定某磁盘有256个柱面，当前磁头位于90号并且向柱面号小的方向移动，其余与上述例子相同，则服务顺序为90-88-72-66-55-0-93-100，移动距离190，磁头改变一次
  • 电梯算法
    • 是对扫描算法的一种改进，允许磁头中途改变方向
    • 在服务过程中总是选择当前磁头方向上最近柱面的I/O操作，在当前方向上没有I/O操作而反方向上有时，立刻改变方向，并选择距离当前磁头方向最近柱面的I/O操作
    • 如上述例子，服务顺序90-88-72-66-55-93-100，移动距离80，磁头改变一次

OS第六章文件管理

发表于 2020-12-12 分类于 课程复习

第6章 文件管理

文件系统的主要功能

  • 文件内容的组织
    • 文件内容的组织形式分为文件的逻辑结构和物理结构
    • 逻辑结构侧重方便用户使用
    • 物理结构侧重提高存储空间的利用率与存取速度
  • 文件和目录管理
  • 文件存储空间管理
  • 文件系统的接口
  • 文件的共享与安全性

文件逻辑结构的分类

  • 流式文件
    • 文件内容按用户提供的数据顺序，以字符流方式组织，没有对内容进行结构上的划分
  • 记录式文件
    • 以数据在逻辑上的完整性含义为单位划分文件内容，每个单位称为一个逻辑记录，简称记录，记录式文件的文件内容是若干个记录的有序集合

三类文件物理结构及其主要特点

  • 连续结构
    • 也称顺序结构，对于一个文件的所有记录，系统按照记录顺序，将它们存放在依次相邻的物理块上，创建或保存一个文件时，系统根据文件的记录数，从外存中查找块数不少于文件记录数的连续空闲物理块
    • 特点
      • 管理简单
      • 存取速度快
      • 存储空间连续分配，存储空间利用率不高
      • 不方便文件内容的增加或删除
  • 链接结构
    • 指定物理块的一个固定存储单元，例如每个块最后一个存储单元，作为链表的指针，指向同一个文件内容存储的下一个物理块，并规定指针为空表示该文件的物理块链表结束
    • 特点
      • 非连续的存储分配，提高了存储空间的利用率
      • 方便文件内容的增加或删除
      • 只适合顺序存取，存取速度慢
      • 指针信息造成物理块信息不完整，并导致数据无法控制
  • 索引结构
    • 类似分页存储管理的页表一样，为外存储器上每一个文件建立一个索引表，登记了该文件的各个记录号（或主键）与物理块的对应关系
    • 特点
      • 非连续的存储分配，提高了存储空间的利用率
      • 方便文件内容的增加或删除
      • 实现随机存取
      • 索引表占用额外的存储空间
      • 增加检索的开销

UNIX多级索引结构

  • UNIX把索引表分成4级，称为多级索引结构
  • UNIX中，每个文件都有一个数据结构，称为i-node，在i-node中定义了13个表项，记为i_addr[0]、i_addr[1]、…、i_addr[12]
    • 其中i_addr[0]~i_addr[9]的10个表项称为直接索引表
    • i_addr[10]为一次间接指针，指向保存一次间接索引表的物理块
    • i_addr[11]为二次间接指针，指向保存二次间接索引表的物理块
    • i_addr[12]为三次间接指针，指向保存三次间接索引表的物理块
  • 假定物理块长度为4KB，每个物理块号用4B表示
    • 这样对于直接索引表等级文件内容存储的前十个物理块，对于文件内容不超过40KB的小文件，只需要i-node直接索引表的10个表项就可以了
    • 当文件内容超40KB时，使用一次间接索引表，一次间接索引表保存文件内容占用的各个物理块号，一个表项表示一个物理块号，占4B，所以一次间接索引表可以表示4KB/4B=1K即1024物理块，文件内容合计为4KB X 1024=4MB，在文件长度超过40KB而小于或等于40KB+4MB时，利用直接索引表和一次间接索引表可以实现对文件的管理
    • 同理，二级索引表就是4GB，三级为4TB
  • 理论上最大可达40KB+4MB+4GB+4TB（针对上述数据）

二级目录结构及访问过程

  • 二级目录结构把目录分为两级：用户文件目录(UFD)和系统主目录(MFD)
  • 访问过程
    • 按照用户名查找系统主目录，得到用户文件目录，接着按照文件名查找用户文件目录，得到文件的FCB，从而得到文件在外存中的存储位置

树状目录结构、根目录、路径含义

  • 根目录
    • 根目录由系统自动建成，用户不能创建或删除
  • 现代操作系统都支持树状目录结构
  • “.”和”..”分别表示当前目录及其父目录

文件保护、保密的含义及基于主体权限的存取控制方式

  • 文件保护
    • 含义
      • 防止文件内容被破坏，这种破坏指存储介质的物理损坏导致文件内容无法读取和用户的误操作造成文件内容的丢失
    • 措施
      • 备份与恢复两种措施
  • 文件保密
    • 含义
      • 文件保密是对文件存取的控制，防止文件的非法访问
    • 文件系统安全的核心是实现文件的保密
  • 基于主体权限的存取控制方式
    • 根据主体对客体所拥有的访问属性而进行存取控制，因此属于自主访问策略（DAC)
    • 主体权限的管理方法主要由三种：存取控制矩阵(PM)，访问控制表(ACL)，权能表(CL)
    • 存取控制矩阵
      • 行表示主体，列表示客体
      • 存取控制矩阵中可能存在大量的空元素或0
    • 存取控制表
      • 是存取控制矩阵的一种简化，为每个客体按列建立一个列表，用于登记可访问的各个用户及其访问属性
    • 权能表
      • 是存取控制矩阵的另一种简化，每个主体按行建立一个列表

OS第五章存储器管理

发表于 2020-12-09 更新于 2020-12-12 分类于 课程复习

第5章 存储器管理

虚拟地址、物理地址、重定位及方式

虚拟地址

  • 程序源代码经过编译、链接后，生成可执行目标程序，在可执行目标程序中，源程序中的每一个变量和指令都在目标程序中都对应一个唯一的编号，这些编号称为变量或指令语句的虚拟地址

物理地址

  • 一道程序在装入内存后，程序的每一个虚拟地址所对应的变量或指令，在内存中都对应唯一的一个存储单元，这个存储单元的地址称为物理地址

重定位

  • 重定位又称地址转换或地址映射，就是把虚拟地址转换为物理地址的过程
  • 程序运行必须重定位的原因————CPU最终是以物理地址存取数据和指令
  • 重定位两种方式
    • 静态重定位
      • 在程序装入时把所有的虚拟地址全部一次地转换为物理地址，在创建相应的进程后，进程运行过程不再需要地址转换
      • 特点：实现简单，不需要硬件支持，但是因为虚拟地址全部转换为物理地址，所以不利于存储保护，并且静态重定位往往要求按照虚拟地址顺序，同一道程序连续占用一次相邻的存储单元，影响存储单元的利用率
      • 单任务的DOS操作系统采用的就是静态重定位
    • 动态重定位
      • 支持多任务的操作系统都采用动态重定位
      • CPU需要访问时才将要访问的指令和数据的虚拟地址转换为物理地址

固定分区：基本思想、数据结构

  • 基本思想
    • 操作系统启动时，把用户区分成若干存储区域，每个区域称为一个分区，各个分区长度可以不等，启动后分区个数与长度不再改变，程序装入时，一个分区只能分配一道程序，一道程序也只能占用一个分区，这种分配也称连续分配
  • 数据结构
    • 固定分区采用的数据结构称为分区说明表，表结构由分区号、起始地址、分区长度和状态组成

可变分区：基本思想、FF、BF和WF分配算法及特点、空闲区回收、对换、覆盖

  • 基本思想

    • 操作系统启动后，整个用户区作为一个空闲区，程序装入时，系统根据程序的实际需求量，查找一个合适的空闲区，如果空闲区长度等于程序实际需求量，就可以直接分配，否则将其划分成两个分区，一个分区分配给当前程序，另一个分区作为空闲区保留下了，进程运行完成撤销后，回收分区，称为空闲区

  • 分配基本策略

    • FF（最先适应法）

      • 从用户区的低地址开始查找，找到能满足程序的第一个空闲区用于分配
      • 特点：小程序集中运行在低地址部分的存储空间上，大程序装入较高地址部分的存储空间上

    • BF(最佳适应法）

      • 查找一个能满足程序需求的，长度最小的空闲区用于分配

      • 特点：尽量把较大的空闲区保留下来，以满足后续大程序的需求，但它将小的空闲区分割成两个分区，使得分配后的剩余空闲区更小，这些小的空闲区不利于将来的程序装入，容易造成碎片

    • WF（最坏适应法）

      • 查找一个能满足程序需求的长度最大的空闲区用于分配
      • 特点：查找大的空闲区用于分配，使得分配后剩余的空闲区尽可能大，有利于其他程序的装入

  • 空闲区回收

    • 在一个进程被撤销时，回收其占用的分区，回收时要判断是否有相邻的空闲区，以便把相邻的空闲区合并成一个大的分区

  • 可变分区的特点

    • 存在碎片，降低了存储空间的利用率
    • 分区个数和每个分区的长度都在变化
    • 为进程的动态扩充存储空间提供可能
    • 需要合并相邻空闲区，增加系统开销
    • 基本分配算法FF、BF、WF在存储空间利用率上没有很大差别

  • 对换、覆盖

    • 在分区管理中，程序大小收到分区长度的限制，因此可以采用对换或覆盖技术
    • 区别
      • 控制不同：对换技术中，进程调入调出由操作系统自动实现，而覆盖是程序员控制
      • 单位粒度不同：对换技术中，调入调出发生在进程之间，整个进程调出或调入，而覆盖技术发生在进程内部，对应子程序之间的调入覆盖
      • 内存扩容的效果不同：在对换中单个进程的地址空间不能超过内存实际大小，而覆盖允许程序大小超过内存大小

分页存储管理中页数计算、虚拟地址寄存器的结构，块长

  • 例如虚拟地址寄存器16位其中低11位表示页内地址，那么块长为2的11次方，即2K，一个进程的最大页数为2的5次方，即64个页，如果虚拟地址寄存器32位，其中低12位表示页内地址，那么块长为2的12次方，即4K，一个进程的最大页数为2的20次方，即1M页

  • 在程序装入的分页过程中，最后不足块长度的程序信息也构成一个页

    • 假定块长2K（2048），有A,B两道程序，大小分别为6872和6016,对于A，按照块长2K分页为4页，最后一个页只包含728个虚拟地址信息，同样，B分为3页，最后一页包含1920个信息

静态分页：基本思想、页表、重定位

  • 基本思想

    • 程序装入时，把程序的所有页全部一次地装入内存，

  • 页表

    • 把登记进程页与对应关系的数据结构称为页表

  • 重定位

    • 分页存储管理采用动态重定位
    • 分页存储管理的重定位过程
      • 页号p与页内地址w：p=虚拟地址/块长，w=虚拟地址%块长，例如B的虚拟地址a=4356，p=2,w=260
      • 存储保护：检查页号p是否小于页表限长寄存器的值，如果大于或等于，则产生地址越界中断
      • 访问页表得到块号：页表起始地址+页表项长度*P
      • 形成物理地址：根据页号p从页表中读取得到对应的b，知道页号p的信息存放在内存中块号b的存储空间上，从而得到物理地址b X 块长 + w，如上述B中a=4356对应的物理地址为326 X 2K + 260

虚拟存储器、程序局部性原理

  • 虚拟存储器思想
    • 没有增加主存储器容量实际大小，而是在软件上采取一些方法，在程序装入时只装入基本的部分信息，其余部分仍然保留在外存中，程序运行中系统调用外存把信息调入主存，使系统可以运行比主存储器容量大的程序
  • 虚拟存储器要解决的主要技术包括理论基础、调入策略、置换算法
  • 程序的局部性原理是虚拟存储器思想的理论基础
  • 程序的局部性原理
    • 分为时间局部性、空间局部性
    • 时间局部性：某个地址的存储单元信息被访问，在不久的将来，这个地址的存储单元信息再次被访问的可能性很大
    • 空间局部性：某一个地址的存储单元被访问后，在不久的将来，与这个地址相邻的存储单元信息被访问的可能性很大

请求分页：扩充页表中P、A、M位的作用，重定位过程缺页中断、页面调度的关系

  • 请求页面实现时关键需要解决的四个问题
    • CPU如何区分内外存的页信息
    • 重定位过程中当发现访问的虚拟地址信息所在的页不在内存时，需要产生特殊的IO操作从外存将其所在的页读入内存
    • 在从外存读一个新的页装入内存时，内存没有空闲块如何处理
    • 程序运行所需的基本页如何确定
  • 解决这四个方面的方法
    • 扩充页表
      • 分页存储管理采用动态重定位，MMU的重定位通过页表实现，因此可以在页表中增加一些实现请求分页所需要的信息
      • 扩充页表中P为中断位、A为访问位，M为修改位
    • 缺页中断及其处理
      • MMU的缺页中断产生后，操作系统响应这个特殊的中断
    • 页面调度
      • 在从外存读一个新的页装入内存时，内存没有空闲块，需要从内存中选择一页将其信息淘汰，空出的块分配给新的页
      • 重定位过程缺页中断、页面调度的关系：在操作系统的缺页中断处理过程中，通过页面调度来分配空出的块给新的页
    • 置换算法
      • 目标：在内存中尽可能保留进程运行过程中经常访问的页，以减少缺页中断的次数
      • 基本的置换算法：先进先出算法(FIFO)，最近最久未使用算法(LRU)，最近最不常用算法（LFU），二次机会置换算法的思想
      • FIFO（先进先出算法）
        • 将内存中的页按装入内存的先后顺序排列，淘汰时选择最先进入内存的页
        • 依据：程序的空间局部性原理
      • LRU（最近最久未使用算法）
        • 页表中登记每个页被CPU访问的时间，淘汰时选择最近一段时间最久没有被访问的页
      • LFU（最近最不常用算法）
        • 页表中登记的每个页被CPU访问的次数，淘汰时选择最近一段时间被访问次数最少的页
      • 二次机会算法
        • 结合页表的访问位A对FIFO置换算法进行改进
        • 在需要淘汰一个页时，选择最先进入内存的页，检查访问位A，如果该页访问位A=0，则淘汰该页，算法结束，如果A=1，将A置为0，并视该页为新装入的而保留在内存中

分段存储管理：基本思想、段表、重定位

  • 基本思想
    • 程序分段
    • 内存动态分区
    • 非连续存储分配
    • 内外村统一管理实现虚拟
  • 数据结构设计
    • 分段存储管理内存中的空闲区采用可变分区的数据结构，如空闲区链表等
    • 在一道程序装入后，需要专门的数据结构来登记它的每个段的信息以及各段在内存中的存放分区位置，这种数据结构称为段表
      • 段表的结构由段号、段长度、中断位P、分区起始地址、外存地址、存取控制信息、访问位A和修改为M组成
      • 每个进程对应一个段表
      • 段表存储在内存的系统区中，作为内核的关键数据结构之一

抖动现象、BELADY现象

  • 抖动现象
    • 如果分配给一个进程的物理页面太少，不能包含整个的工作集，即常驻集包含于工作集，那么进程将会造成很多的缺页中断，需要频繁地在内存与外存之间替换页面，从而使进程的运行速度变得很慢，把这种状态称为“抖动”。
    • 产生抖动的原因：随着驻留内存的进程数目增加，分配给每个进程的物理页面数不断减小，缺页率不断上升。所以OS要选择一个适当的进程数目和进程需要的帧数，以便在并发水平和缺页率之间达到一个平衡。
  • BELADY现象
    • Belady现象：在采用FIFO算法时，有时会出现分配的物理页面数增加，缺页率反而提高的异常现象。
    • 原因：FIFO算法的置换特征与进程访问内存的动态特征是矛盾的，与置换算法的目标是不一致的（即替换较少使用的页面），因此，被它置换出去的页面并不一定是进程不会访问的。

分段和分页主要区别

  • 存储空间的分配单位粒度
    • 在分页存储管理中以页为单位，页由硬件虚拟地址结构决定，页长度固定
    • 在分段存储管理中以段位单位，段由程序员程序设计，段之间长度往往不等
  • 虚拟地址空间的维数
    • 分页中虚拟地址空间一维
    • 分段存储管理中虚拟地址二维
  • 内存分配
    • 分页存储管理把内存空间看成由一组大小相等的块组成
    • 分段存储管理采用动态分区
  • 碎片
    • 在分页存储管理中，每个进程的最后一个页可能不足一个块的长度，按页分配内存块时，存在内碎片
    • 而分段存储管理则采用动态分区，随着分配和回收的进行，可能存在很小的空闲区，造成外碎片

段页式存储管理：基本思想

  • 基本思想
    • 内存分块
      • 采用分页管理中的内存分块思想，把内存看成一系列固定长度的块组成，每个块对应一个块号
    • 程序分段
      • 程序由若干个在逻辑上具有完整独立意义的单位组成，每个单位称为段，系统在链接程序时，为每个段的信息建立独立的虚拟地址空间
    • 段分页
      • 类似分页管理中的进程分页思想
    • 非连续的分配
      • 以页为单位分配内存块，同一个段的几个相邻的页在内存中不要求占用相邻的内存块，同一个页的程序信息在内存中是连续存放，不同页之间内存中的程序信息可以是不连续的
    • 实现虚拟存储器
      • 在装入一个程序时，可以只装入一部分段的基本页，其余保留在外存中（虚拟存储器内容部分1.6）

段页式存储管理中进程、段表和页表的关系

  • 每个进程由多个段组成，每个段又分为若干页

OS第四章进程调度与死锁

发表于 2020-12-09 更新于 2020-12-12 分类于 课程复习

第4章 进程调度与死锁

批处理系统中作业的四个状态

  • 提交状态
  • 后备状态
  • 执行状态
  • 完成状态

作业调度：含义、基本算法（FCFS，SJF，HRN）、周转时间及平均周转时间的计算

作业调度含义

  • 按一定的策略从后备队列中选择一部分作业，为他们分配运行所需的必要资源，创建进程的过程

作业调度基本算法

  • 先来先服务算法(FCFS)
    • FCFS（First Come First Serviced)
    • 按作业提交时间的先后顺序，调度时选择最先提交的作业
    • 特点
      • 公平合理
      • 算法简单，容易实现
      • 服务质量欠佳
    • FCFS作业调度算法对长作业有利，而不利于短作业，这里的长短是指作业在运行过程中占用处理器时间的多少
  • 短作业优先算法(SJF)
    • SJF(Shortest Job First)
    • 每个作业都要求给定其运行所需的处理器时间，调度时选择一个拥有最小处理器时间的作业
    • 特点
      • 算法思想简单，但实现困难
      • 拥有最小的平均周转时间
      • 吞吐量大
      • 存在“饥饿”现象（较长的作业可能迟迟没有机会被选中，一直推迟）
  • 高响应比优先算法(HRN)
    • HRN(Highest Response_ratio Next)
    • 每个作业定义一个响应比，调度时先计算后备队列中各作业的响应比，选择具有最大响应比的一个作业
    • 一个作业的响应比R=作业等待时间/作业大小（其中作业等待时间=系统当前时间-作业提交时刻）
    • 高响应比优先算法综合考虑了FCFS和SJF，但因为需要事先知道作业的大小，所以和SJF一样实现比较困难

进程调度：含义、基本算法（FCFS，RR，优先级）、周转时间及平均周转时间的计算

进程调度含义及两种方式

  • 按一定策略从进程就绪队列中选择一个进程，让其占用处理器运行
  • 两种基本的进程调度方式
    • 抢占方式：可以让运行中的进程停下来
    • 非抢占方式：除非进程由于自身或外部原因，否则不会终止它

进程调度基本算法

  • 先来先服务算法(FCFS)
    • 属于非抢占式方式
    • 按照进入就绪队列的先后顺序，调度时选择最先进入就绪队列的进程，这种调度算法称为先来先服务算法FCFS
    • 特点
      • 简单容易实现
      • 公平合理
      • 服务质量欠佳
  • 短进程优先算法（与作业调度大同小异）
  • 时间片轮转算法(RR)
    • RR(Round Robin)
    • 就绪队列按先进先出方式组织，调度时选择一个进程，并分配一个时间片，选中的进程在规定时间片内运行
    • 运行时出现的三种情况
      • 时间片用完，进程任务还没完成，则进程必须让出处理器，回到就绪队列末尾，进行下一轮调度
      • 时间片未用完，进程任务结束，立即引起新调度，当前定时器还剩多少时间，新的调度都不予考虑，重新分配一个新的时间片
      • 时间片未用完，进程进入阻塞状态，将在被唤醒时加入到就绪队列末尾
      • I/O不划分时间片，CPU内才划分
      • 假设就绪队列中的进程数为n，时间片为T,那么响应时间R为R=T * n
      • 特别当系统中所有进程需要的处理器时间都小于系统的时间片时，RR算法相当于FCFS算法
  • 优先级算法
    • 每个进程给予一个优先数，调度时选择优先级最高的进程
    • 设计优先级时可以分为静态优先级和动态优先级

死锁：产生原因，解决方法及其特点

  • 死锁产生的根本原因：系统拥有的资源数量小于各进程对资源的需求总数
  • 死锁的四个必要条件
    • 互斥条件
    • 不剥夺条件
    • 请求与保持条件
    • 环路等待条件
  • 死锁解决方法
    • 预防
      • 死锁预防是指在资源分配上采取一些限制措施来破坏死锁产生的四个必要条件之一
      • 破坏请求与保持条件，在系统中不允许进程在已获得某种资源的情况下，申请其他资源。即要想出一个办法，阻止进程在持有资源的同时申请其他资源。
      • 缺点：资源被严重浪费，严重恶化了资源的利用率，使进程经常会发生饥饿现象。
    • 避免
      • 预防没有找到理想办法，因此将资源分配的限制减弱，这就是死锁避免
      • 死锁避免的关键是系统安全状态检查，目前具有代表性的是银行家算法
      • 银行家算法参考资料：银行家算法
    • 检测与恢复
      • 在死锁避免仍然没有找到理想算法的情况下，鉴于死锁产生可能性极小，提出死锁检测与恢复的办法
      • 检测：安排一个检测程序，定期运行，检查系统是否存在进程死锁，如果发现某些进程处于死锁，则设法采取措施解决死锁
      • 恢复：可采取剥夺资源或撤销进程或重新启动系统的方法

OS第三章进程管理

第3章 进程管理

多道程序设计、并发执行宏观、微观理解

  • 宏观上，多道程序“同时”在运行，表现为多任务
  • 微观上，任何时刻至多只有一道程序真正在运行中

并发执行的工作方式及特征

  • 并发执行的工作方式
    • 处理器在执行一道程序的两条指令之间，可以执行其他程序的指令
  • 特征
    • 随机性
      • 在多道程序中，哪一道程序先运行，哪一道后运行，运行时能连续运行多长时间都是随机的
    • 不可再现性
    • 相互制约
      • 在微观上，一道程序运行时其他程序不能运行

进程定义、特征，进程与程序的主要区别及联系，进程状态及转换（动态性）

  • 进程定义
    • 一道程序在一个数据集上的一次执行过程，称为一个进程
  • 进程特征
    • 动态性
    • 并发性
    • 独立性
    • 结构性
    • 异步性
  • 进程与程序的主要区别与联系
    • 主要区别：进程是个动态过程，程序是个静态实体
    • 联系
      • 进程是程序的运行过程，因此进程包含了程序
      • 进程的运行就是对应程序的运行
      • 程序规定了进程要完成的功能
  • 进程的基本状态
    • 运行
    • 就绪
    • 阻塞
  • 进程转换

进程的表示（PCB）及作用

  • 操作系统把为了描述、管理、控制进程所设计的数据结构称为进程控制块(PCB），创建一个进程就是为其创建一个PCB
  • 作用：PCB是进程存在的标识，操作系统通过PCB管理控制进程

信号量机制及实现互斥和同步的方法，生产者/消费者问题并发程序设计

  • 信号量机制

    • 信号量机制是一种可以实现进程互斥、同步的有效方法，信号量其实就是一个变量（可以是一个整数，也可以是更复杂的记录型变量），可以用一个信号量来表示系统中某种资源的数量。如系统中只有一台打印机，就可以设置一个初值为1的信号量。如所有的临界资源在同一时刻只能被一个进程访问，所以临界资源的信号量为1。

  • 实现互斥与同步的方法

    • 用户进程可以通过使用操作系统提供的一对原语（P\V）来对信号量进行操作，从而很方便的实现了进程的互斥和同步。

    • 关于P\V原语

      • p操作

      • v操作

    • 信号量实现互斥

    • 信号量并发程序设计

  • 生产者/消费者问题(PC问题)

    • 解法1：假设生产者进程和消费者进程不能同时访问缓冲区buf，此时二者不仅具有一般同步关系，还具有互斥关系 在该解法中，将缓冲区视为临界资源，但在实际情况中，可以不用考虑生产者与消费者的互斥关系，因而有了解法2
    • 解法2：对每一个缓冲区单元格buf[x]设置空或满的标记

进程通信的含义

  • 两个或多个进程之间交换数据的过程称为进程通信，其中提供数据的一方成为发送进程，得到数据的一方称为接收进程

进程通信的实现方式

  • 共享存储区通信
    • 建立一个共享的存储区域，通信时发送进程与接受进程都可以访问该区域
  • 消息缓冲通信
    • 消息缓冲通信是最基本的进程通信
    • 消息缓冲通信的设计主要包括数据结构设计和发送、接受操作的设计，其中数据结构主要有消息缓冲区的结构、PCB的通信参数结构
    • 消息缓冲通信是一种直接通信（如果接收进程尚未被创建，则通信不成功，并且消息缓冲区队列建立在接收进程内部，通信缺乏灵活性）
    • 只能应用在同一台计算机（由于send()要把消息缓冲区直接加入接收进程对应的消息缓冲区队列中，因为双方进程必须在同一内存中）
  • 信箱通信
    • 信箱通信是应用最广泛的进程通信
  • 管道通信
    • 利用文件系统的功能，通信双方的进程共享同一个文件，发送进程向文件中写数据，接收进程从文件中读数据

线程的引入、类型、与进程的关系

  • 线程的含义
    • 把进程细化成若干个可以独立运行的实体，每一个实体称为一个线程
  • 引入线程的目的
    • 实现进程内部的并发执行，提高并行程度
    • 减少处理器切换带来的开销
    • 简化进程通信方式
  • 线程的类型
    • 用户级线程
      • 运行在用户空间，内核感觉不到用户线程的存在，内核管理的仍然是进程，这也是其最大的优点，即统一进程的线程之间的处理器切换不必进入内核，只需在用户态进行，大大减少了处理器切换带来的开销，
      • 但同时也影响想了并行程度的提高（如果因系统调用而阻塞，线程所在的进程整个都会被阻塞）
    • 系统级线程
      • 优缺点与用户级刚好相反
  • 线程与进程的关系
    • 根本区别：线程是处理器分配调度的基本单位，进程是其他资源（既处理器之外）分配的基本单位
    • 线程与进程一样，都具有生命期，也具有并发性和三个基本状态（运行、就绪、阻塞），具有相互制约关系的线程之间也需要同步、互斥控制

OS第二章操作系统接口

发表于 2020-12-08 更新于 2020-12-12 分类于 课程复习

第2章 操作系统接口

OS内核

  • 内核主要组成
    • 与硬件密切相关的操作
    • 关键数据结构
    • 基本中断处理程序
    • 使用频繁的功能模块
  • 内核的基本特点
    • 常驻内存
      • 可以理解成缓存，程序完成后仍保留在内存中，下次运行就不再需要从磁盘上读入内存
    • 运行在核心态
      • 内核程序运行在核心态

用户接口类型

  • 命令接口
    • 外部命令
      • 如果一个命令对应一个独立的应用程序文件，这种命令称为外部命令
    • 内部命令
      • 由内核程序模块实现的命令称为内部命令
  • 程序接口
    • 程序接口是通过系统调用来实现的

系统调用含义

  • 系统调用含义
    • 一组操作系统设计人员事先编写好的子程序，并作为内核的一部分
    • 程序员使用这组子程序的方法
  • 关于系统调用的实现过程
    • （1）处理器切换为核心态，保护用户程序的现场
    • （2）分析功能号并在地址入口表中查找对应的子程序，有时还需要进行安全控制检查
    • （3）执行系统调用的子程序并得到结果
    • （4）现场恢复，处理器切换为用户态并返回结果，必要时进行安全检查

系统调用与用户子程序的主要区别

  • 系统调用运行在核心态，而用户子程序运行在用户态
  • 系统调用的执行产生中断，用户子程序的运行则不会产生中断
  • 系统调用子程序的代码与调用者的程序代码是分开独立的，而用户子程序代码与调用者的程序代码在同一个进程地址空间
  • 不同用户程序可以共享使用同一个系统调用，用户子程序通常不能由其他用户程序调用

OS定义

  • 通常把计算机硬件系统上配置的第一个大型软件称为计算机操作系统

多道程序设计

  • 多道程序设计概念：在内存中同时存放多道程序，这些程序可以并发执行

多道程序设计并发执行与现代操作系统的关系

  • 人们对多道程序并发执行工作方式的复杂性和存在问题的研究及解决，形成了操作系统原理的核心内容

操作系统形成的基础

  • 硬件上具有中央处理器与设备，设备与设备并行工作的能力
  • 软件上提出了多道程序设计技术

批处理系统工作的四个阶段、JCL

  • 批处理系统工作的四个阶段
    • 提交
    • 后备
    • 执行
    • 完成
  • JCL（作业控制语言）
    • 是由操作系统提供的一组命令，程序员用操作系统提供的JCL命令定义作业、描述对作业的运行控制

SPOOLing技术及组成

  • 输入井（充当输入磁带）
  • 预输入程序（充当输入计算机）
  • 输出井（充当输出磁带）
  • 缓输出程序（充当输出计算机）

并发/并行执行的含义

  • 并发执行概念：在多道程序设计环境下，处理器在开始执行一道程序的第一条指令后，在这道程序完成之前，处理器可以开始执行另一道程序甚至更多的其他程序（这种工作流程的外在表现就是多任务）
  • 好处：多道程序的并发执行，可以提高CPU的利用率

OS主要功能

  • 用户接口及作业管理

  • 存储器管理
  • 文件系统

Java的==与equal

发表于 2020-12-06 更新于 2020-12-12 分类于 Java学习

equals介绍

  • JAVA当中所有的类都是继承于Object这个超类的，在Object类中定义了一个equals的方法，这个方法的初始行为是比较对象的内存地址，但在一些类库当中这个方法被复写了，如String、Integer、Date。在这些类当中equals有其自身的实现，而不再是比较类在堆内存中的存放地址了。
  • 所以说，对于复合数据类型之间进行equals比较，在没有覆写equals方法的情况下，他们之间的比较还是内存中的存放位置的地址值，跟双等号（==）的结果相同；如果被重写，按照重写的要求来。
  • 总而言之，当equals没有被重写时，实际上就和==是等价的

“==”介绍

  • 基本数据类型中”==”的比较比较的是他们的值
  • 而在复合数据类型（类）中，比较的是他们在内存中的存放地址
  • 对于第二种类型，除非是同一个new出来的对象，他们的比较后的结果为true，否则比较后结果为false。因为每new一次，都会重新开辟堆内存空间。
  • 举例说明

在上述第四行中，用”==”比较str1和str2，返回的结果却是false；而用“equals”比较str1和str2，返回的结果是true，而str3为str2的引用传递，此时”==”的结果就是true。

面试题

请解释字符串比较之中“==”和equals()的区别？

“==”：比较的是两个字符串内存地址的数值是否相等，属于数值比较；
equals()：比较的是两个字符串的内容，属于内容比较。
以后进行字符串相等判断的时候都使用equals()。
注意这里说的是字符串比较中，equals其实已经重写了
