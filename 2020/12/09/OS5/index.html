第5章 存储器管理

虚拟地址、物理地址、重定位及方式

虚拟地址

  • 程序源代码经过编译、链接后，生成可执行目标程序，在可执行目标程序中，源程序中的每一个变量和指令都在目标程序中都对应一个唯一的编号，这些编号称为变量或指令语句的虚拟地址

物理地址

  • 一道程序在装入内存后，程序的每一个虚拟地址所对应的变量或指令，在内存中都对应唯一的一个存储单元，这个存储单元的地址称为物理地址

重定位

  • 重定位又称地址转换或地址映射，就是把虚拟地址转换为物理地址的过程
  • 程序运行必须重定位的原因————CPU最终是以物理地址存取数据和指令
  • 重定位两种方式
    • 静态重定位
      • 在程序装入时把所有的虚拟地址全部一次地转换为物理地址，在创建相应的进程后，进程运行过程不再需要地址转换
      • 特点：实现简单，不需要硬件支持，但是因为虚拟地址全部转换为物理地址，所以不利于存储保护，并且静态重定位往往要求按照虚拟地址顺序，同一道程序连续占用一次相邻的存储单元，影响存储单元的利用率
      • 单任务的DOS操作系统采用的就是静态重定位
    • 动态重定位
      • 支持多任务的操作系统都采用动态重定位
      • CPU需要访问时才将要访问的指令和数据的虚拟地址转换为物理地址

固定分区：基本思想、数据结构

  • 基本思想
    • 操作系统启动时，把用户区分成若干存储区域，每个区域称为一个分区，各个分区长度可以不等，启动后分区个数与长度不再改变，程序装入时，一个分区只能分配一道程序，一道程序也只能占用一个分区，这种分配也称连续分配
  • 数据结构
    • 固定分区采用的数据结构称为分区说明表，表结构由分区号、起始地址、分区长度和状态组成

可变分区：基本思想、FF、BF和WF分配算法及特点、空闲区回收、对换、覆盖

  • 基本思想

    • 操作系统启动后，整个用户区作为一个空闲区，程序装入时，系统根据程序的实际需求量，查找一个合适的空闲区，如果空闲区长度等于程序实际需求量，就可以直接分配，否则将其划分成两个分区，一个分区分配给当前程序，另一个分区作为空闲区保留下了，进程运行完成撤销后，回收分区，称为空闲区

  • 分配基本策略

    • FF（最先适应法）

      • 从用户区的低地址开始查找，找到能满足程序的第一个空闲区用于分配
      • 特点：小程序集中运行在低地址部分的存储空间上，大程序装入较高地址部分的存储空间上

    • BF(最佳适应法）

      • 查找一个能满足程序需求的，长度最小的空闲区用于分配

      • 特点：尽量把较大的空闲区保留下来，以满足后续大程序的需求，但它将小的空闲区分割成两个分区，使得分配后的剩余空闲区更小，这些小的空闲区不利于将来的程序装入，容易造成碎片

    • WF（最坏适应法）

      • 查找一个能满足程序需求的长度最大的空闲区用于分配
      • 特点：查找大的空闲区用于分配，使得分配后剩余的空闲区尽可能大，有利于其他程序的装入

  • 空闲区回收

    • 在一个进程被撤销时，回收其占用的分区，回收时要判断是否有相邻的空闲区，以便把相邻的空闲区合并成一个大的分区

  • 可变分区的特点

    • 存在碎片，降低了存储空间的利用率
    • 分区个数和每个分区的长度都在变化
    • 为进程的动态扩充存储空间提供可能
    • 需要合并相邻空闲区，增加系统开销
    • 基本分配算法FF、BF、WF在存储空间利用率上没有很大差别

  • 对换、覆盖

    • 在分区管理中，程序大小收到分区长度的限制，因此可以采用对换或覆盖技术
    • 区别
      • 控制不同：对换技术中，进程调入调出由操作系统自动实现，而覆盖是程序员控制
      • 单位粒度不同：对换技术中，调入调出发生在进程之间，整个进程调出或调入，而覆盖技术发生在进程内部，对应子程序之间的调入覆盖
      • 内存扩容的效果不同：在对换中单个进程的地址空间不能超过内存实际大小，而覆盖允许程序大小超过内存大小

分页存储管理中页数计算、虚拟地址寄存器的结构，块长

  • 例如虚拟地址寄存器16位其中低11位表示页内地址，那么块长为2的11次方，即2K，一个进程的最大页数为2的5次方，即64个页，如果虚拟地址寄存器32位，其中低12位表示页内地址，那么块长为2的12次方，即4K，一个进程的最大页数为2的20次方，即1M页

  • 在程序装入的分页过程中，最后不足块长度的程序信息也构成一个页

    • 假定块长2K（2048），有A,B两道程序，大小分别为6872和6016,对于A，按照块长2K分页为4页，最后一个页只包含728个虚拟地址信息，同样，B分为3页，最后一页包含1920个信息

静态分页：基本思想、页表、重定位

  • 基本思想

    • 程序装入时，把程序的所有页全部一次地装入内存，

  • 页表

    • 把登记进程页与对应关系的数据结构称为页表

  • 重定位

    • 分页存储管理采用动态重定位
    • 分页存储管理的重定位过程
      • 页号p与页内地址w：p=虚拟地址/块长，w=虚拟地址%块长，例如B的虚拟地址a=4356，p=2,w=260
      • 存储保护：检查页号p是否小于页表限长寄存器的值，如果大于或等于，则产生地址越界中断
      • 访问页表得到块号：页表起始地址+页表项长度*P
      • 形成物理地址：根据页号p从页表中读取得到对应的b，知道页号p的信息存放在内存中块号b的存储空间上，从而得到物理地址b X 块长 + w，如上述B中a=4356对应的物理地址为326 X 2K + 260

虚拟存储器、程序局部性原理

  • 虚拟存储器思想
    • 没有增加主存储器容量实际大小，而是在软件上采取一些方法，在程序装入时只装入基本的部分信息，其余部分仍然保留在外存中，程序运行中系统调用外存把信息调入主存，使系统可以运行比主存储器容量大的程序
  • 虚拟存储器要解决的主要技术包括理论基础、调入策略、置换算法
  • 程序的局部性原理是虚拟存储器思想的理论基础
  • 程序的局部性原理
    • 分为时间局部性、空间局部性
    • 时间局部性：某个地址的存储单元信息被访问，在不久的将来，这个地址的存储单元信息再次被访问的可能性很大
    • 空间局部性：某一个地址的存储单元被访问后，在不久的将来，与这个地址相邻的存储单元信息被访问的可能性很大

请求分页：扩充页表中P、A、M位的作用，重定位过程缺页中断、页面调度的关系

  • 请求页面实现时关键需要解决的四个问题
    • CPU如何区分内外存的页信息
    • 重定位过程中当发现访问的虚拟地址信息所在的页不在内存时，需要产生特殊的IO操作从外存将其所在的页读入内存
    • 在从外存读一个新的页装入内存时，内存没有空闲块如何处理
    • 程序运行所需的基本页如何确定
  • 解决这四个方面的方法
    • 扩充页表
      • 分页存储管理采用动态重定位，MMU的重定位通过页表实现，因此可以在页表中增加一些实现请求分页所需要的信息
      • 扩充页表中P为中断位、A为访问位，M为修改位
    • 缺页中断及其处理
      • MMU的缺页中断产生后，操作系统响应这个特殊的中断
    • 页面调度
      • 在从外存读一个新的页装入内存时，内存没有空闲块，需要从内存中选择一页将其信息淘汰，空出的块分配给新的页
      • 重定位过程缺页中断、页面调度的关系：在操作系统的缺页中断处理过程中，通过页面调度来分配空出的块给新的页
    • 置换算法
      • 目标：在内存中尽可能保留进程运行过程中经常访问的页，以减少缺页中断的次数
      • 基本的置换算法：先进先出算法(FIFO)，最近最久未使用算法(LRU)，最近最不常用算法（LFU），二次机会置换算法的思想
      • FIFO（先进先出算法）
        • 将内存中的页按装入内存的先后顺序排列，淘汰时选择最先进入内存的页
        • 依据：程序的空间局部性原理
      • LRU（最近最久未使用算法）
        • 页表中登记每个页被CPU访问的时间，淘汰时选择最近一段时间最久没有被访问的页
      • LFU（最近最不常用算法）
        • 页表中登记的每个页被CPU访问的次数，淘汰时选择最近一段时间被访问次数最少的页
      • 二次机会算法
        • 结合页表的访问位A对FIFO置换算法进行改进
        • 在需要淘汰一个页时，选择最先进入内存的页，检查访问位A，如果该页访问位A=0，则淘汰该页，算法结束，如果A=1，将A置为0，并视该页为新装入的而保留在内存中

分段存储管理：基本思想、段表、重定位

  • 基本思想
    • 程序分段
    • 内存动态分区
    • 非连续存储分配
    • 内外村统一管理实现虚拟
  • 数据结构设计
    • 分段存储管理内存中的空闲区采用可变分区的数据结构，如空闲区链表等
    • 在一道程序装入后，需要专门的数据结构来登记它的每个段的信息以及各段在内存中的存放分区位置，这种数据结构称为段表
      • 段表的结构由段号、段长度、中断位P、分区起始地址、外存地址、存取控制信息、访问位A和修改为M组成
      • 每个进程对应一个段表
      • 段表存储在内存的系统区中，作为内核的关键数据结构之一

抖动现象、BELADY现象

  • 抖动现象
    • 如果分配给一个进程的物理页面太少，不能包含整个的工作集，即常驻集包含于工作集，那么进程将会造成很多的缺页中断，需要频繁地在内存与外存之间替换页面，从而使进程的运行速度变得很慢，把这种状态称为“抖动”。
    • 产生抖动的原因：随着驻留内存的进程数目增加，分配给每个进程的物理页面数不断减小，缺页率不断上升。所以OS要选择一个适当的进程数目和进程需要的帧数，以便在并发水平和缺页率之间达到一个平衡。
  • BELADY现象
    • Belady现象：在采用FIFO算法时，有时会出现分配的物理页面数增加，缺页率反而提高的异常现象。
    • 原因：FIFO算法的置换特征与进程访问内存的动态特征是矛盾的，与置换算法的目标是不一致的（即替换较少使用的页面），因此，被它置换出去的页面并不一定是进程不会访问的。

分段和分页主要区别

  • 存储空间的分配单位粒度
    • 在分页存储管理中以页为单位，页由硬件虚拟地址结构决定，页长度固定
    • 在分段存储管理中以段位单位，段由程序员程序设计，段之间长度往往不等
  • 虚拟地址空间的维数
    • 分页中虚拟地址空间一维
    • 分段存储管理中虚拟地址二维
  • 内存分配
    • 分页存储管理把内存空间看成由一组大小相等的块组成
    • 分段存储管理采用动态分区
  • 碎片
    • 在分页存储管理中，每个进程的最后一个页可能不足一个块的长度，按页分配内存块时，存在内碎片
    • 而分段存储管理则采用动态分区，随着分配和回收的进行，可能存在很小的空闲区，造成外碎片

段页式存储管理：基本思想

  • 基本思想
    • 内存分块
      • 采用分页管理中的内存分块思想，把内存看成一系列固定长度的块组成，每个块对应一个块号
    • 程序分段
      • 程序由若干个在逻辑上具有完整独立意义的单位组成，每个单位称为段，系统在链接程序时，为每个段的信息建立独立的虚拟地址空间
    • 段分页
      • 类似分页管理中的进程分页思想
    • 非连续的分配
      • 以页为单位分配内存块，同一个段的几个相邻的页在内存中不要求占用相邻的内存块，同一个页的程序信息在内存中是连续存放，不同页之间内存中的程序信息可以是不连续的
    • 实现虚拟存储器
      • 在装入一个程序时，可以只装入一部分段的基本页，其余保留在外存中（虚拟存储器内容部分1.6）

段页式存储管理中进程、段表和页表的关系

  • 每个进程由多个段组成，每个段又分为若干页
